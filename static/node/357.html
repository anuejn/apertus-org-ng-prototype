AXIOM Beta Progress Report II

Submitted by Sebastian on Mon, 12/01/2014 - 13:30

We've just finished the Schematic and Layout for three
new AXIOM Beta Boards and updated the Sensor Board. \o/

The purpose of the Beta Test Board is to allow testing
integration between MicroZed and the Interface Boards
with various external power supplies and level shifters.
It can be considered a specialized breakout board.

The Dummy Interface Board allows us to connect the Sensor
Board with the Beta Board without intermediate processing.
Naturally this limits the number of LVDS channels similar
to what we did on the AXIOM Alpha, but it should allow
to get a complete system up and running soon.

Note: we replaced the FX10-120/12 with an FX10-100/10
for mechanical reasons with an option to have a separate
power connector if the current design proves too noisy.

The CMV12000 Sensor Board was improved based on the
feedback we got and we also created a ZIF socket version
which will hopefully allow to test sensors more easily.
(we use an Intel Xeon 604 type socket for this purpose)

All Schematics and Layouts as well as some fancy pictures
are available below.

Again, if you are experienced with schematics and/or
board layout, please take the time to review the design
and notify us about anything you find.

An idea was to put a "small" FPGA onto the image sensor board so we can create a unified interface between any image sensor and our actual image processing pipeline running on the Microzed. This way every image sensor can be taught to speak the same language and that takes us a bit closer to implementing easily swappable image sensor modules. Instead of putting this FPGA onto the image sensor module itself, we have now settled for a setup that does this in two modules/steps. An Image Sensor holder board plus an Image Sensor Interface Board.
