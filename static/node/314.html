AXIOM Beta (版)



加入早期用户社区，
订阅AXIOM Beta资讯，加入邮件列表：


AXIOM Beta版本比AXIOM Alpha版更小巧、更便宜，流线型设计更适合拍摄，测试和研发。
这次我们制定了一个覆盖软件和硬件开发领域的众筹计划，希望得到你们的力量推动研发。只要验证了设计方案的可行性，我们会计划第一批制作大约250台摄像机，而这批机器将专门为这次资金捐助者提供。每一台机器都有个性化定制的空间。


AXIOM Beta版本计划包括的特性：

  • HDMI全高清（4:4:4）输出，高达每秒60帧
  • 以实验性的HDMI格式进行4K RAW数据输出
  • 向MicroSD卡写入全分辨率，全采样精度的原始静态图像
  • 可以从手机、平板电脑及笔记本上遥控操作所有摄像机的功能
  • 电源管理和监控（比如：电压、电流、温度）
  • 通过模块化的I/O插件实现高度定制（比如：串行数字接口SDI)
  • 安装加速度计，磁力计和陀螺仪，比如：用以保证图像稳定
  • 不同的镜头卡口（如尼康F-mount，EF和M4/3卡口）
  • 宽输入电压范围（5-40V）
  • 设计轻巧紧凑～110x60x50mm
  • 嵌入式Linux系统（比如 Raspian, ArchLinux）
  • 颜色映射表格（LUTs），矩阵颜色转换（matrix color conversion），FPN补偿（FPN compensation），伪彩色显示（false color display），图像叠加（overlays），坏点补偿（dead pixel compensation）
  • 使用Microzed控制面板（替代了 AXIOM Alpha使用的 Zed控制面板）

为谁打造AXIOM Beta摄像机？

有一些公司欺骗消费者为beta产品支付较高的价格，从而最终毁掉了整个开发项目。这不是我们的作风，我们一定要明确的让消费者知道将会拿到的是什么样的产品。事实上，我们希望让消费者从一开始就知道自己得到的是什么。AXIOM Beta是一个beta阶段的产品（从名字就可以看出来）。机器肯定会有一些漏洞和问题，但是我们确信，有了你的帮助，你的测试，以及（但不是最重要的）你的金钱，我们会将每一个漏洞穷追到底，并将它改造成一个优势特性。

像往常一样，AXIOM Beta版第一天就能完美工作的可能性很小，同样我们也不能保证开始它一定不会运行失败。没有什么是完美的，至少现在AXIOM Beta还不是完美的，但这是我们的目标，在所有方面都运转顺利之前我们还需要大量的开发时间和固件升级。
我们诚邀您加入我们的探索，不是作为一个旁观者，而是真正为塑造最终产品贡献一份力量。为了回报您的帮助，我们会为您的硬件购买提供折扣，这将引出我们下一个话题：

AXIOM Beta的制作成本是多少？

参加AXIOM Beta版本众筹项目的资助者，出资金额在350欧元或者485美元以上的，最终只需支付制造成本（比如材料和装配费用），大概在550欧元或者760美元左右（不含摄像传感器，具体看下表），就可以获得一台AXIOM Beta摄像机（包含所有必须的I/O模块插件）。
但请放心，不管你选择什么硬件，你都只需要支付材料费、组装费和运费。假如我们能够吸引到更多的人加入这个融资计划，每个人就能花更少的钱买到相同的产品。这个融资计划的最主要的目的是为了找到早期的使用者，以尽快将硬件送至黑客及开发者手中。一旦产品成熟了，你可以预见AXIOM摄像机会的功能会的越来越强大，价格也会的走高。基于这个原因， 未来的AXIOM 模型机器的价格不太可能像这次众筹计划中的AXIOM Beta价格这么低。


升级路径

假如你决定将AXIOM Beta 升级到下一个系列产品（比如AXIOM Gamma，假如我们持续用希腊字母进行命名的话），它的主要组件（也是最昂贵的）传感器很可能被重复使用。Microzed也是一个独立的组件，并可以被其他项目使用或者被再次售卖。


图像传感器的选择

刚开始，我们想基于Truesense KAC1204传感器建立 AXIOM Beta，因为我们想使用一款不同于AXIOM Alpha摄像机中用到的传感器，以积累更多的经验。一旦这样可行，我们就可以根据我们社区会员的兴趣点加入更多的图像传感器选项。这个传感器可能来自于Cmosis公司，Truesense公司或者一个完全不同的制造商，只要他们提供数据表并使用兼容的接口，我们就会考虑。请于下方进行投票，帮助我们做出让您满意的选择。
